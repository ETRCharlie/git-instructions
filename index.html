GitStuff

A respository ("repo") is a project. A branch is a small offshoot of the main ("master") repo that will likely be deleted after it is committed back to the repo. A fork is a separate copy of that repo which will likely live on its own.

The basic idea is: Create (or get access to) a repo at GitHub, copy it to your local machine ("clone"), branch that repo, make your changes locally to the new branch, add that branch to the system ("add"), let GitHub know you have changes ready to move into production ("commit"), push the branch into production ("push"), merge your branch into the master ("merge").

First, download and install Github Desktop app, and/or download and install the command-line app. Install both as your own user, not as admin.

Create a location in your local system where you want to have all your Git-related fun. If you're using the command-line app, change to that location within that command-line app.

In GitHub.com, go to your repo page, click the "clone or download" button and copy a repo to your local system (there is a command-line option for "git clone RepoURL, but just downloading the files from here is easier). OR, if your files are already local, use GitHub.com to create a new repo and upload those files to the new repo.

Make a branch of the repo, work on that branch. You can switch between the branch and the master to view differences/changes in both the command-line app (git checkout YourBranchName) or the desktop app. When your edits are done and the branch has been merged, submit a Pull Request.

Before Edits

git clone GitRepoURL
downloads a copy of the master

git checkout -b YourNewBranchName
creates a new (-b) branch of the repo

git branch
tells you which branch you're working on

git checkout SomeBranchName
switches you to that branch

git status
shows you the state of the git branch

After Edits

git add file.name
stages that file for committing

git add *
stages all the files for committing

git commit -m "message with description of changes"
commits that file to the repo

git commit -a -m "adds and commits all files"
adds ALL files and commits them
(combines previous two commands)

git push --set-upstream origin YourNewBranchName
moves your branch up to GitHub

git merge YourBranchName
merges your branch with the master (?)
